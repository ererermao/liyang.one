¶5.1 Too many arguments (first extra is ‘start’).

删除profile的配置，

¶5.2 ExecStart=/etc/rc.d/init.d/mysqld start (code=exited, status=1/FAILURE)

直接用命令执行，可以看到更详细的日志

¶5.3 mysqld_safe error: log-error set to ‘/var/log/mariadb/mariadb.log’, however file don’t exists. Create writable for user ‘mysql’.

touch /var/log/mariadb/mariadb.log && chmod 777 /var/log/mariadb/mariadb.log

¶5.4 os_file_get_status() failed on ‘./ibdata1’. Can’t determine file permissions or InnoDB: ./ib_logfile0 can’t be opened in read-write mode

目录的权限问题，就先简单粗暴解决吧，chwon -R 777 /opt/mysql
